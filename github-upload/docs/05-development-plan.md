# Development Plan

## 외부 API

| 기능 | 사용 목적 | 외부 API | 우선순위 |
| --- | --- | --- | --- |
| 회원가입/로그인 | Google 계정 인증 | Google OAuth 2.0 | 필수 |
| 전국 도서관 조회 | 공공도서관 목록, 주소, 운영 정보 | 도서관 정보나루 `libSrch` | 필수 |
| ISBN 도서 조회 | ISBN/바코드 기반 도서 정보 | 도서관 정보나루 `srchDtlList` | 필수 |
| 도서 소장 여부 조회 | 특정 도서관의 도서 보유 여부 | 도서관 정보나루 `bookExist` | 선택 |
| 추천 도서 조회 | 인기 대출 도서 및 추천 도서 | 도서관 정보나루 `srchBooks` | 선택 |
| 주소 좌표 변환 | 도서관 주소를 위도/경도로 변환 | Kakao Local API | 보조 |
| 주변 도서관 검색 | GPS 기준 도서관 탐색 보조 | Kakao Local API | 보조 |
| 감상평 AI 검증 | 감상평과 도서 내용 관련성 검증 | Gemini API + 로컬 규칙 | 필수 |

## 환경 변수

```env
KAKAO_JS_API_KEY=

```

## API 계약 확정 필요 사항

| 항목 | 결정 필요 |
| --- | --- |
| 인증 방식 | JWT Access/Refresh Token vs 현재 쿠키 세션 |
| 주변 도서관 query | `latitude/longitude` vs 현재 구현의 `lat/lng` |
| AI 검증 실행 시점 | submit 직후 자동 실행 vs 별도 complete API |
| 세션 ping 주기 | 예: 30초, 60초 |
| 위치 이탈 실패 기준 | 1회 이탈 경고, N회 이상 실패, 연속 시간 기준 |
| EXP/영향력 계산 | 고정 10점 vs 독서 시간/페이지 기반 |
| 랭킹 갱신 | 실시간 계산 vs 캐시 테이블 배치 갱신 |

## MVP 완료 기준

- Google 로그인 후 사용자 정보가 저장된다.
- 신규 사용자는 닉네임과 진영을 설정할 수 있다.
- 지도에서 주변 도서관과 점령 상태를 볼 수 있다.
- 도서관 250m 이내에서만 독서 세션을 시작할 수 있다.
- ISBN으로 도서 정보를 등록할 수 있다.
- 20분 이상 독서 후 인증 정보를 제출할 수 있다.
- AI와 시스템 검증 결과에 따라 성공/실패가 결정된다.
- 성공 시 EXP, 영향력, 점령권, 기록이 갱신된다.
- 마이페이지에서 인증 성공 도서와 세션 기록을 확인할 수 있다.
