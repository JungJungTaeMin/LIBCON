# User Flow

## 전체 흐름

```text
앱 실행
→ Google 로그인
→ 신규 사용자/온보딩 완료 여부 확인
→ 닉네임 입력
→ 닉네임 중복 확인
→ 진영 선택 또는 진영 생성
→ 서비스 이용 안내
→ 메인 지도 진입
→ 현재 위치 기반 주변 도서관 조회
→ 도서관 상세 확인
→ 반경 250m 이내 체크인
→ ISBN/바코드 스캔
→ 도서 정보 등록
→ 독서 세션 시작
→ 최소 20분 이상 독서
→ 독서 중 위치 확인
→ 시작 페이지 / 종료 페이지 입력
→ 감상평 작성
→ 인증 정보 제출
→ 감상평 AI 검증
→ 페이지 수 검증
→ 위치 검증
→ 최종 인증 통과
→ 사용자 EXP 지급
→ 소속 진영 영향력 증가
→ 도서관 점령권 갱신
→ 개인/진영 랭킹 반영
→ 마이페이지에서 기록 확인
```

## 1. Google 로그인

1. 사용자가 앱을 실행한다.
2. 스플래시 이후 로그인 화면으로 이동한다.
3. 사용자가 Google 로그인 버튼을 누른다.
4. Google OAuth 인증을 완료한다.
5. 클라이언트 또는 OAuth callback 서버가 Google 사용자 정보를 확인한다.
6. 서버는 기존 가입자 여부와 온보딩 완료 여부를 확인한다.
7. 온보딩 완료 사용자는 메인 지도 화면으로 이동한다.
8. 신규 사용자 또는 온보딩 미완료 사용자는 온보딩 화면으로 이동한다.

### 로그인 데이터

| 데이터 | 설명 |
| --- | --- |
| socialProvider | `google` |
| socialId | Google 고유 사용자 ID |
| name | Google 계정 이름 |
| email | Google 계정 이메일 |

## 2. 온보딩

1. 닉네임을 입력한다.
2. 형식과 중복 여부를 검증한다.
3. 기존 진영 목록을 조회한다.
4. 기존 진영을 선택하거나 새 진영을 생성한다.
5. 서비스 이용 규칙을 확인한다.
6. 서버는 사용자 프로필과 진영 정보를 저장한다.
7. 온보딩 완료 후 메인 지도 화면으로 이동한다.

### 닉네임 검증

| 항목 | 기준 |
| --- | --- |
| 길이 | 2자 이상 12자 이하 |
| 허용 문자 | 한글, 영문, 숫자 |
| 공백 | 앞뒤 공백 제거 |
| 중복 | 기존 사용자 닉네임과 중복 불가 |

## 3. 지도 및 도서관

- 기본 조회 반경은 5km다.
- 도서관 핀 색상은 `libraries.current_occupied_faction_id` 기준으로 표시한다.
- 미점령 도서관은 기본 색상으로 표시한다.
- 도서관이 너무 많으면 지도 확대/축소에 따라 클러스터링한다.
- 주변 도서관이 없으면 10km 확장 안내를 제공한다.
- 위치 권한이 거부되면 기본 지역 지도를 표시하고 권한 요청을 안내한다.

## 4. 독서 세션

1. 사용자가 도서관 상세 화면에서 독서 시작을 누른다.
2. 앱은 현재 GPS 위치와 정확도를 확인한다.
3. 서버는 도서관 반경 250m 이내 여부를 검증한다.
4. 사용자가 ISBN/바코드를 스캔한다.
5. 서버는 도서 정보를 조회하거나 외부 API에서 가져와 저장한다.
6. 세션 상태를 `IN_PROGRESS`로 저장하고 타이머를 시작한다.
7. 독서 중 앱은 위치를 주기적으로 서버에 전송한다.
8. 20분 이상 독서하면 인증 제출이 가능해진다.

### 세션 상태

| 상태 | 설명 |
| --- | --- |
| `IN_PROGRESS` | 독서 타이머 작동 중 |
| `READY_TO_VERIFY` | 최소 시간이 지나 인증 제출 가능 |
| `SUBMITTED` | 인증 정보 제출 완료 |
| `VERIFICATION_PASSED` | 최종 인증 성공 |
| `VERIFICATION_FAILED` | 최종 인증 실패 |
| `CANCELED` | 위치 이탈 또는 사용자 종료로 중단 |

## 5. 인증 및 보상

### 제출 항목

| 항목 | 설명 |
| --- | --- |
| startPage | 시작 페이지 |
| endPage | 종료 페이지 |
| reviewText | 감상평 |

읽은 페이지 수는 직접 입력하지 않고 계산한다.

```text
읽은 페이지 수 = 종료 페이지 - 시작 페이지 + 1
```

### 최종 인증 통과 기준

```text
1. 감상평 검증 통과
2. 페이지 수 검증 통과
3. 위치 검증 통과
4. 최소 20분 독서 조건 충족
```

### 보상 흐름

```text
최종 인증 성공
→ 사용자 EXP 증가
→ 소속 진영의 도서관 영향력 증가
→ 도서관 점령권 갱신
→ 개인 랭킹 및 진영 랭킹 반영
```

인증 실패 시 EXP와 영향력은 지급하지 않는다.

## 6. 점령권 갱신

1. 인증 성공 후 사용자의 소속 진영 영향력이 증가한다.
2. 해당 도서관의 모든 진영 영향력 점수를 비교한다.
3. 가장 높은 영향력을 가진 진영을 확인한다.
4. 기존 점령 진영과 다르면 `libraries.current_occupied_faction_id`를 갱신한다.
5. 영향력 증가와 점령권 변경 여부를 `influence_logs`에 저장한다.

동점일 경우 기존 점령 진영을 유지한다.
